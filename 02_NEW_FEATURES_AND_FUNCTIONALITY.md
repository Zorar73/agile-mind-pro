# Agile Mind Pro — Новые фичи и функционал

## Концепция: Два типа пользователей

**Проблема:** Есть офисные сотрудники, которые работают с полным функционалом (задачи, доски, спринты). А есть розничные сотрудники (продавцы, консультанты), которым нужны только:
- Новости компании
- Обучение (курсы, тесты)
- Возможно: чат, справочники, мотивация

**Решение:** Гибкая система ролей с доступом к модулям. Админ создаёт роли и определяет, какие разделы видит каждая роль.

---

## 1. Система ролей и доступа к модулям

### 1.1 Модули системы

Разбиваем весь функционал на независимые модули:

| Модуль | Описание | Для кого |
|--------|----------|----------|
| **Задачи** | Kanban, создание задач, подзадачи | Офис |
| **Доски** | Управление досками, колонками | Офис |
| **Спринты** | Планирование, burndown | Офис |
| **Календарь** | Дедлайны, события | Офис |
| **Наброски** | Sketch → Task, AI конвертация | Офис |
| **Команды** | Управление участниками | Офис |
| **Новости** | Лента новостей компании | Все |
| **Обучение** | Курсы, уроки, тесты | Все |
| **Чат** | Общение внутри команды | Все |
| **Справочники** | Документы, инструкции, FAQ | Все |
| **Мотивация** | Рейтинги, достижения, бонусы | Розница |
| **Профиль** | Личные данные, настройки | Все |

---

### 1.2 Конструктор ролей

**Админ создаёт роли и выбирает:**
- Название роли (например: "Продавец", "Офис-менеджер", "Руководитель командаа")
- Какие модули доступны
- Права внутри каждого модуля (просмотр / редактирование / администрирование)

**Примеры готовых ролей:**

| Роль | Доступные модули |
|------|------------------|
| Офис (полный) | Все модули |
| Руководитель розницы | Новости, Обучение, Чат, Справочники, Мотивация, Профиль |
| Продавец | Новости, Обучение, Чат, Справочники, Мотивация, Профиль |
| Стажёр | Новости, Обучение, Профиль |

---

### 1.3 Права внутри модулей

**Пример для модуля "Обучение":**
- **Просмотр** — проходить курсы, смотреть свой прогресс
- **Редактирование** — создавать курсы, добавлять уроки
- **Администрирование** — назначать курсы, смотреть статистику всех, управлять категориями

**Пример для модуля "Новости":**
- **Просмотр** — читать новости, ставить реакции
- **Редактирование** — создавать новости
- **Администрирование** — удалять, закреплять, модерировать

---

### 1.4 Назначение ролей

**Варианты назначения:**
- Вручную каждому пользователю
- Массово — выбрать несколько человек → назначить роль
- При регистрации — роль по умолчанию для новых

**Организационная структура:**
- Регионы → командаы → Сотрудники
- Роль можно назначать на уровне структуры (все продавцы региона "Центр")

---

### 1.5 UI настройки ролей

**Страница "Роли и доступ" в админке:**
- Список ролей с количеством пользователей
- Создание новой роли
- Матрица: модули × права (чекбоксы)
- Предпросмотр — как выглядит интерфейс для этой роли

**При назначении роли пользователю:**
- Мгновенное изменение интерфейса
- Боковое меню показывает только доступные разделы
- Попытка зайти на недоступную страницу → редирект на главную

## 2. LMS — Система обучения

### 2.1 Структура контента

```
Категории
└── Курсы
    └── Модули (разделы курса)
        └── Уроки
            ├── Теория (текст, видео, картинки)
            ├── Тест
            └── Практическое задание
---


### 2.2 Назначение обучения

**Обязательные курсы:**
- Назначаются на роль/должность
- Дедлайн прохождения
- Напоминания (за 3 дня, за 1 день, просрочено)
- Эскалация руководителю при просрочке

**Добровольные курсы:**
- Открыты для самостоятельного изучения
- Каталог курсов с поиском и фильтрами

**Автоназначение:**
- При приёме на работу (роль "Стажёр" → курс "Адаптация")
- При смене должности
- Периодическое (раз в год — compliance)

---

### 2.4 Прогресс 

**Прогресс пользователя:**
- Начатые / В процессе / Завершённые курсы
- Процент прохождения каждого курса
- Результаты тестов (балл, попытки)
- Время, потраченное на обучение

**Сертификаты:**
- Автоматическая генерация при завершении курса
- Галерея сертификатов в профиле

---

### 2.5 Аналитика обучения

**Для сотрудника:**
- Мои курсы, прогресс, сертификаты
- Рейтинг среди коллег (опционально)
- Рекомендуемые курсы

**Для админов**
- Общая статистика по компании
- Прохождение по регионам/командаам
- Сложные вопросы (где чаще ошибаются)
- Эффективность курсов (correlation с показателями)
- Экспорт отчётов

---

### 2.6 Геймификация обучения

**Достижения:**
- "Первый курс" — завершил первый курс
- "Отличник" — 100% по тесту
- "Марафонец" — 5 курсов за месяц
- "Эксперт по продукту" — все курсы категории
---

## 3. Новости и коммуникации

### 3.1 Лента новостей

**Типы публикаций:**
- Новость (текст + фото/видео)
- Объявление (важное, закреплённое сверху)
- Опрос
- Поздравление (дни рождения, юбилеи)

**Таргетинг:**
- На всю компани
- На команду
- На роль (только продавцы)

**Реакции и комментарии:**
- Лайки, emoji-реакции
- Комментарии (с @упоминаниями)
- Ответы на комментарии

**Обязательное прочтение:**
- Отметка "Прочитал" для важных новостей
- Статистика просмотров
- Напоминание непрочитавшим

## 6. Обратная связь

### 6.1 Кнопка "Сообщить о проблеме"

**Типы обращений:**
- Ошибка в системе
- Предложение по улучшению
- Вопрос по обучению
- Другое

**Автоматически собирается:**
- Страница, где возникла проблема
- Браузер, устройство
- Скриншот (опционально)

**Обработка:**
- Поступает в админку
- Назначается ответственный
- Статус: Новое → В работе → Решено
- Ответ пользователю

---

### 6.2 Оценка контента

**После прохождения курса:**
- Оценка курса (1-5 звёзд)
- Что понравилось / что улучшить (опционально)

**Анонимные опросы:**
- Удовлетворённость (eNPS)
- Обратная связь по процессам
